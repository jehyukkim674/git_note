# git_note — 설계 문서

작성일: 2026-06-20

## 개요

안드로이드와 Mac에서 모두 동작하는 크로스플랫폼 메모장 앱. 메모는 마크다운(`.md`) 파일로
저장되고 HTML 라이브 미리보기를 제공한다. 기기 간 동기화는 **GitHub 저장소(진짜 git)**를
통해 이루어진다. 소수의 사용자가 같은 저장소를 공유하며 협업하는 용도.

## 목표 / 사용 시나리오

- 사용 범위: 소수 공유/협업 (가족·팀원 몇 명이 GitHub 저장소 하나를 공유).
- 메모를 안드로이드/Mac에서 보고 편집하며, GitHub로 자동 동기화.
- 오프라인 편집 후 온라인 복귀 시 자동 동기화.

## 기술 구성

- **Tauri v2** — Rust 백엔드 + React(TypeScript) 프론트엔드.
- **하나의 React 코드베이스**로 Mac 데스크톱 앱 + 안드로이드 앱 빌드.
- **편집기**: CodeMirror 6 (모바일 입력 지원 양호).
- **미리보기**: markdown-it로 HTML 렌더링 + DOMPurify sanitize.
- **Git**: Rust `git2`(libgit2), **HTTPS 전송만** 사용. Tauri command로 프론트에 노출.
- **인증**: GitHub OAuth **device flow**. 토큰은 OS 보안 저장소(Mac Keychain / Android
  Keystore)에 저장.

## 데이터 모델 (메모 ↔ 파일 매핑)

- GitHub 저장소 1개 = "보관함(vault)". 앱 데이터 폴더에 clone.
- 메모 = `.md` 파일. 폴더 = 디렉토리 (저장소 디렉토리 구조 그대로 매핑).
- 이미지/첨부 = `assets/` 등의 폴더에 파일로 저장하고 상대경로로 참조 → 같이 커밋.
- 앱 설정(대상 저장소, 브랜치 등)은 로컬에 저장.

## 컴포넌트 (각각 독립적으로 테스트 가능)

1. **git-core (Rust)**: clone / pull / commit / push / status / 충돌 감지. Tauri command.
2. **auth (Rust)**: OAuth device flow, 토큰 저장·조회.
3. **vault 서비스 (TS)**: 트리 목록, 노트 읽기/쓰기, 검색.
4. **editor (React)**: CodeMirror + 미리보기.
5. **UI 셸**: 사이드바 트리 · 검색 · 편집/미리보기 영역.

각 컴포넌트는 명확한 인터페이스로만 통신하며, 내부 구현을 바꿔도 소비자가 깨지지 않도록 한다.

## UI 레이아웃

데스크톱(Mac): 3분할 — 좌측 폴더 트리/검색 | 중앙 편집기 | 우측 미리보기(토글 가능).

모바일(안드로이드): 스택 내비게이션 — 노트 목록(트리/검색) → 탭하면 편집기로 이동,
상단에 미리보기 토글.

## 동기화 흐름

- **앱 열 때 / 포커스 복귀 시**: fetch + pull (fast-forward 또는 merge).
- **저장 시**: 파일 쓰기 → `git add` → 자동 커밋 → push.
  - push 거절되면 → pull(merge) 후 재시도.
  - 진짜 충돌이면 → 해당 노트를 "충돌" 상태로 표시 + 충돌 해결 화면 제공.
- **오프라인**: 로컬 커밋만 쌓아두고 온라인 복귀 시 push. "동기화 대기" 상태 표시.

## 검색

로컬에 clone되어 있으므로 v1은 `.md` 파일들을 즉석 grep(제목·본문)으로 처리. 향후 인덱싱 도입 가능.

## 에러 처리

- 네트워크 실패 → 오프라인 상태 표시, 온라인 복귀 시 자동 동기화.
- 토큰 만료 → 재인증 안내.
- 병합 충돌 → 충돌 해결 UI.
- push 거절 → 자동 pull 후 재시도.

## 테스트 전략

- **Rust**: git-core를 임시 저장소(temp repo) 대상으로 단위 테스트.
- **TS**: vault 서비스 + 검색 로직 테스트.
- **컴포넌트**: 편집기/미리보기 렌더링 테스트.

## 기술 리스크 및 백업 플랜

**최대 리스크: 안드로이드에서 libgit2 빌드.** 데스크톱은 쉽지만, 안드로이드는
`git2`(libgit2 + TLS)를 NDK로 크로스컴파일해야 한다 (`cargo-ndk` + vendored libgit2,
HTTPS 전용). 까다롭지만 가능한 영역.

**백업 플랜**: 안드로이드 git 빌드가 막히면, 안드로이드만 **GitHub API(HTTP)** 방식으로
대체한다 (데스크톱은 진짜 git 유지). git-core 인터페이스를 추상화해 두면 플랫폼별 구현 교체가
가능하다.
